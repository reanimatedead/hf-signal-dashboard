# Data Contract

Integration schema between **fx-analysis-system** (analysis engine) and **hf-signal-dashboard** (public dashboard).

---

## Overview

Two repositories form one market analysis stack:

```
fx-analysis-system  →  signals.json / signals.csv  →  hf-signal-dashboard
(local analysis)           (integration contract)         (public dashboard)
```

This document defines:
1. The **native format** (`data.json`) used internally by hf-signal-dashboard
2. The **integration format** (`signals.json`) for fx-analysis-system to supply data

---

## 1. Native format: data.json

Generated by `fetch_signals.py` via GitHub Actions. Consumed directly by `docs/index.html`.

**File location**: `docs/data.json`

**Top-level structure**:

```json
{
  "meta": {
    "updated_at": "2026-06-03T08:05:50+09:00",
    "updated_at_str": "2026-06-03 08:05 JST",
    "next_update_str": "2026-06-04 08:00 JST",
    "elapsed_sec": 51.8,
    "counts": {
      "nikkei225": 142,
      "dow30": 30,
      "nasdaq100": 90,
      "sp500": 102,
      "fx": 18
    }
  },
  "summary": {
    "nikkei225": { "BUY": 3, "HOLD": 88, "WATCH": 44, "AVOID": 7 }
  },
  "markets": {
    "nikkei225": [ { "symbol": "...", "signal": "HOLD", ... } ],
    "dow30": [ ... ],
    "nasdaq100": [ ... ],
    "sp500": [ ... ],
    "fx": [ ... ]
  }
}
```

**Per-symbol fields (equities)**:

| Field | Type | Description |
|---|---|---|
| `symbol` | string | Ticker (e.g. `7203.T`, `AAPL`) |
| `name` | string | Display name |
| `price` | number | Last close price |
| `change_pct` | number | Daily change % |
| `rsi` | number | RSI(14) |
| `macd_hist` | number | MACD histogram |
| `macd_direction` | string | `"UP"` or `"DOWN"` |
| `ema20/50/200` | number | EMA values |
| `ema_signal` | string | `"BULLISH"`, `"BEARISH"`, `"MIXED"` |
| `bb_pct` | number | Bollinger %B (0–1) |
| `composite_score` | integer | 0–100 composite signal score |
| `signal` | string | `STRONG BUY`, `BUY`, `HOLD`, `WATCH`, `AVOID` |

**Per-symbol fields (FX)**:

| Field | Type | Description |
|---|---|---|
| `signal` | string | `STRONG LONG`, `LONG`, `NEUTRAL`, `SHORT`, `STRONG SHORT` |
| `cci_daily` | number | CCI(288) daily |
| `cci_4h` | number | CCI(288) 4-hour |
| `mc_probability` | number | Monte Carlo probability (0–1) |

## 2. Integration format: signals.json

Produced by **fx-analysis-system** for cross-system data supply.
This format is the integration contract between the two repositories.

**File location (source)**: `exports/signals.json` in fx-analysis-system  
**File location (target)**: `docs/sample-signals.json` in hf-signal-dashboard (sample / fallback)

### Schema

```json
{
  "generated_at": "2026-06-03T00:00:00+09:00",
  "source": "fx-analysis-system",
  "timeframe": "1d",
  "version": "1.0",
  "signals": [
    {
      "symbol": "USDJPY",
      "name": "USD/JPY",
      "market": "FX",
      "timeframe": "1d",
      "signal": "watch",
      "score": 72,
      "trend": "up",
      "risk": "medium",
      "comment": "Momentum is positive but volatility risk remains."
    }
  ]
}
```

### Required fields

| Field | Type | Description |
|---|---|---|
| `generated_at` | ISO 8601 string | Generation timestamp with timezone |
| `source` | string | Always `"fx-analysis-system"` |
| `timeframe` | string | Data timeframe (e.g. `"1d"`, `"4h"`) |
| `version` | string | Schema version (`"1.0"`) |
| `signals` | array | One object per symbol |

### Per-signal required fields

| Field | Type | Allowed values |
|---|---|---|
| `symbol` | string | Ticker or pair (e.g. `"USDJPY"`, `"GOLD"`) |
| `market` | string | `"FX"`, `"Equity"`, `"Commodity"`, `"Index"` |
| `signal` | string | See signal values below |
| `score` | integer | 0–100 |
| `trend` | string | `"up"`, `"down"`, `"neutral"` |
| `risk` | string | `"low"`, `"medium"`, `"high"` |
| `comment` | string | Plain-text analysis note (no buy/sell language) |

### Signal values

| Value | Meaning |
|---|---|
| `strong_watch` | Strong attention warranted |
| `watch` | Monitoring recommended |
| `neutral` | No directional signal |
| `caution` | Risk factors present |
| `avoid` | High-risk environment |

### Risk values

| Value | Meaning |
|---|---|
| `low` | Normal conditions |
| `medium` | Elevated conditions |
| `high` | Stress conditions |

---

## Forbidden fields

The following must never appear in signals.json:

- API keys or tokens
- Trading account information
- Order IDs or position sizes
- `buy` / `sell` / `long` / `short` as signal values
- Personal information
- Broker credentials
